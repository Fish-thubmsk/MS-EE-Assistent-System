## 1. Router Agent - 意图识别

### 1.1 LLM 意图分类 Prompt

**文件位置**: `agents/router_agent.py` (第265-275行)

**当前 Prompt**:
```
你是一个意图识别助手，帮助识别用户想要进入哪种学习模式。

模式说明：
- quiz（刷题模式）：用户想要做题、练习、批改或讲解题目
- qa（问答模式）：用户想了解某个知识点、概念，或查阅笔记
- diagnosis（诊断模式）：用户想了解自己的学习情况、薄弱项或获得学习建议

仅回答以下之一（小写英文）：quiz / qa / diagnosis / unknown

用户输入：{user_input}
意图：
```

**功能**: 三分类问题（quiz / qa / diagnosis / unknown）

**优化空间**:
- [ ] 增加子模式识别的例子（如 follow_up、grading 等）
- [ ] 提升置信度计算的准确性
- [ ] 增加困难样本的处理（如混合场景）

**关键词规则** (第107-135行):
- QUIZ 关键词: 做题、刷题、练习、题目、批改、讲解等
- QA 关键词: 什么是、解释、定义、概念、原理等
- DIAGNOSIS 关键词: 诊断、分析、薄弱、错题率、学情等

## 2. Quiz Agent - 刷题模式

### 2.1 批改 (Grading) Prompt

**文件位置**: `agents/quiz_agent.py` (第297-310行)

**当前 Prompt**:
```
你是严谨公正的老师，请批改学生答案。
题型：{question_type}
题目：{content}
参考答案：{correct_answer}
学生答案：{user_answer}

请以 JSON 格式返回批改结果，不要包含任何其他内容：
{"is_correct": true/false, "score": 0-100, "feedback": "详细批改意见"}

评分规则：
- is_correct 为 true 表示答案正确或语义等价，false 表示有明显错误
- score 为 0-100 的整数，支持部分得分（答对要点可得部分分）
- 语义等价的答案（如同义词、等价表达）应判为正确
- feedback 需包含：评分理由、错误点分析（如有）、改进建议
```

**输出格式**: 严格 JSON
```json
{
  "is_correct": true/false,
  "score": 0-100,
  "feedback": "详细批改意见"
}
```

**优化空间**:
- [ ] **题型感知**: 不同题型（选择题/填空/主观题）有不同规则，需细化
- [ ] **分步得分**: 复杂题目分步骤评分（如证明题分论证过程、结论等）
- [ ] **知识点映射**: 自动识别错误涉及的知识点
- [ ] **反馈质量**: 改进意见要针对性强，说明错在哪
- [ ] **0-100分数映射**: 明确各分数段的定义

### 2.2 解析 (Explanation) Prompt

**文件位置**: `agents/quiz_agent.py` (第450-458行)

**当前 Prompt**:
```
你是专业辅导老师，请为以下题目给出详细解析和学习建议。

题目：{content}
参考答案：{correct_answer}
题目解析：{analysis}
批改意见：{feedback}（得分：{score}）
[可选] 参考资料：{ctx_str}

请给出：
1. 详细解题思路
2. 关键知识点
3. 易错点提醒
4. 学习建议
```

**优化空间**:
- [ ] **解题步骤**: 补充中间步骤推导，不仅给结论
- [ ] **知识深度**: 根据题型深度调整讲解层次
- [ ] **关联知识**: 链接相关知识点，形成知识网络
- [ ] **易错分析**: 统计常见错误模式，针对性提醒
- [ ] **学习资源**: 推荐相关公式、定理、例题

### 2.3 追问生成 (Followup) Prompt

**文件位置**: `agents/quiz_agent.py` (第547-553行)

**当前 Prompt**:
```
作为学生，请根据老师的解析生成2-3个追问问题，帮助深入理解。

题目：{content}
本次结果：{status}
老师解析：{explanation[:500]}

请每行输出一个追问，不加序号和前缀。
```

**输出**: 每行一个追问（2-3个）

**优化空间**:
- [ ] **答对时追问**: 问更难的变体、推广、反例
- [ ] **答错时追问**: 问基础概念、为什么这个错法不对
- [ ] **部分正确时追问**: 问缺漏的要点、相关知识
- [ ] **追问链**: 构建逐步深入的追问链路，不仅一问一答

## 3. RAG Agent - 问答模式

### 3.1 问答生成 Prompt

**文件位置**: `agents/rag_agent.py` (第115-129行)

**当前 Prompt**:
```
你是一个专业的学习助手，请根据以下检索到的参考资料，为用户的问题提供清晰、准确的答案。

参考资料：
{context}

用户问题：{question}

要求：
1. 答案中需引用参考资料，用 [序号] 标注（如 [1]、[2]）
2. 若参考资料中有题目，可附上正确答案
3. 如有相关知识点，可适当补充说明
4. 支持中英文混合回答
5. 若参考资料不足以回答问题，请说明并给出基于已有知识的回答

答案：
```

**优化空间**:
- [ ] **检索覆盖**: 确保检索到的资料足够回答问题（如需更多结果）
- [ ] **引用格式**: 检验引用是否准确对应参考资料位置
- [ ] **知识补充**: 何时补充教科书知识 vs 仅基于检索结果
- [ ] **推荐相关**: 在答案末尾推荐相关知识点、题目、笔记
- [ ] **错误纠正**: 检索结果有误时如何处理

## 4. Diagnosis Agent - 诊断模式

### 4.1 诊断报告生成逻辑

**文件位置**: `agents/diagnosis_agent.py` (第470-529行)

**当前逻辑** (非 LLM Prompt，而是模板生成):
```
1. 掌握程度总结（按知识点列出准确率）
2. 薄弱点列表（准确率 < 60%，分高/中优先级）
3. 针对性学习建议与推荐题目
```

**输出示例**:
```
============================================================
📊 学习诊断报告
============================================================
用户：user_001
分析范围：全科

【当前掌握程度总结】
  ✅ 数学::极限：准确率 80.0%（4/5）
  ⚠️ 数学::导数：准确率 50.0%（2/4）

【知识薄弱点列表】
  🔴 [高优先] 数学 - 导数：准确率 50.0%（4 次）

【针对性学习建议与推荐题目】
  📝 推荐练习题：
    • (ID=123) 数学 [中等] 求 f(x)=x²在x=1处的导数…
============================================================
```

**优化空间**:
- [ ] **动态阈值**: 根据科目、年级动态调整 60% 阈值
- [ ] **知识图谱**: 推荐题目时考虑知识点间的依赖关系
- [ ] **学习路径**: 生成个性化复习计划（按难度递进）
- [ ] **对标数据**: 展示学生与平均水平的对比
- [ ] **预测模型**: 预测该学生最可能的薄弱点

## 5. 优化建议

### 5.1 按优先级排列的优化方向

#### 🔴 P0 - 立即优化（影响准确率）

1. **Quiz Agent - 批改 Prompt**
   - **问题**: 批改准确度低，特别是部分得分逻辑混乱
   - **优化**: 
     - 按题型分类详细规则（选择题/填空/主观题各不同）
     - 明确 0-100 分数阶梯（如 90-100 为完全正确，60-80 为关键要点对）
     - 增加中英文混合题、符号数学题的处理
   - **时间**: 1-2 小时
   - **测试**: 准备 10-20 个不同题型的测试案例

2. **Quiz Agent - 解析 Prompt**
   - **问题**: 解析不够深入，常数学题只给答案不给步骤
   - **优化**:
     - 要求完整的推导过程（特别是数学证明题）
     - 明确解题思路的三个层次：基础概念 → 方法选择 → 具体计算
     - 针对常见错误补充反面教学
   - **时间**: 1-2 小时

#### 🟡 P1 - 次优先（增强体验）

3. **RAG Agent - 答案生成**
   - **问题**: 检索很好但答案生成质量不稳定
   - **优化**:
     - 增加 "若检索资料不足，说明缺口" 的明确指导
     - 补充教科书基础知识的标准表述
     - 在答案末尾自动推荐 2-3 个相关知识点

4. **Diagnosis Agent - 诊断报告**
   - **问题**: 报告内容较模板化，缺乏个性化深度
   - **优化**:
     - 根据错题模式生成针对性建议（如 "你的错误多在计算步骤" vs "概念理解不清"）
     - 推荐题目时体现递进关系（从基础到进阶）
     - 添加成长预测（如 "按此计划复习，预计 7 天内提升 10%"）

#### 🟢 P2 - 可选优化（长期）

5. **Router Agent - 意图识别**
   - **问题**: 混合意图、模糊意图识别准确度一般
   - **优化**: 增强多意图检测、子模式识别

6. **系统级优化**
   - 各 Agent 间的提示词协调（如诊断推荐题目时复用 Quiz 的批改规则）
   - 多轮对话的上下文管理

### 5.2 优化工作流

#### Step 1: 测试案例准备（2 小时）
```
为每个 Prompt 准备 10-20 个测试案例：
- 批改 Prompt: 各题型的标准答案、部分答案、错误答案各 5 个
- 解析 Prompt: 简单/中等/困难各 5 个
- 问答 Prompt: 知识点查询、题目查询、混合查询各 5 个
```

#### Step 2: A/B 对比测试（2-3 小时）
```
1. 运行当前 Prompt，记录结果
2. 改进 Prompt，运行新版本
3. 对比：准确率、响应时间、用户满意度
```

#### Step 3: 迭代优化（持续）
```
- 周一：集中优化 P0 项（批改 + 解析）
- 周二-三：A/B 测试 + 数据收集
- 周四：调整 + 部署新版本
- 周五：评估 + 计划下周
```

### 5.3 Prompt 编写最佳实践

#### 通用原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **明确角色** | 告诉 LLM 扮演什么角色 | "你是严谨公正的老师" |
| **清晰任务** | 说明要做什么 | "请批改学生答案" |
| **具体规则** | 给出评分/输出规则 | "分为 0-100，支持部分得分" |
| **格式指定** | 明确输出格式 | "返回 JSON 格式：{...}" |
| **示例展示** | 最好给出 1-2 个例子 | "示例：{"is_correct": true, ...}" |
| **容错处理** | 说明边界情况如何处理 | "若参考答案缺失，输出 None" |

#### 数学/逻辑题特别处理

```
对数学题，要求：
1. 明确列出每一步推导
2. 标注使用的公式/定理
3. 说明为什么这样做（而不是那样做）
4. 给出易错点提醒

对逻辑题，要求：
1. 分析前提、论点、结论
2. 检查逻辑链条是否完整
3. 指出常见的逻辑谬误
```
