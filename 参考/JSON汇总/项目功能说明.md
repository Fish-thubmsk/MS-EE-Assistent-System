# 考研刷题小项目功能说明

## 1. 项目目标

基于现有 MySQL 题库，提供一个可直接使用的网页刷题系统，支持按科目、题型、年份随机抽题，并完成作答与判题交互。


## 2. 已实现功能

### 2.1 后端能力（Flask + MySQL）

- 提供科目接口：`/api/subjects`
- 提供题型接口：`/api/subject/<subject>/types`
- 提供年份接口：`/api/years/<subject>`
- 提供刷题接口：`/api/question/<subject>/<qtype>?year=YYYY`
- 题目从数据库随机抽取，支持年份过滤
- 返回完整题干/文章（避免被截断）
- 针对英语阅读/新题型/完形返回小题号字段 `item_no`


### 2.2 前端能力（HTML + CSS + JS）

- 科目切换：政治 / 数学 / 英语
- 题型切换：按科目动态加载
- 年份筛选：支持“全部年份”与具体年份
- 点击“下一题”随机换题
- 题目卡片样式美化（选项 A/B/C/D 徽标化展示）
- 长文本展示优化（不截断，自动换行）


### 2.3 数学公式渲染

- 集成 MathJax（tex-svg）
- 支持 `$...$` 与 `$$...$$` 公式渲染
- 解决题干公式显示不完整/渲染失败问题


### 2.4 作答与判题交互

- 单选题：单选后点击“提交答案”判题
- 多选题：支持多次勾选/取消，再点击“提交答案”判题
- 判题后展示：
  - 参考答案
  - 用户选择
  - 正确/错误结果
- 视觉反馈：
  - 正确选项高亮（绿色）
  - 错误选项高亮（红色）
  - 已提交题目锁定，防止反复改答案


### 2.5 英语题型展示规则

- 阅读理解 / 新题型 / 完形填空：题干区域展示为“小题号”（如“第 4 题”）
- 完形填空：显示 A/B/C/D 选项
- 文章内容作为阅读材料完整展示


## 3. 当前项目结构（核心）

- `quiz_app.py`：Flask 后端与题目查询接口
- `templates/index.html`：页面结构
- `static/style.css`：页面样式与交互视觉
- `static/app.js`：前端逻辑（刷题、筛选、判题）


## 4. 已知限制

- 数学答案在源数据中多数缺失，部分题型可能无标准答案
- 个别年份在不同科目中的覆盖不一致（由原始数据决定）
- 当前未实现用户维度功能（错题本、收藏、做题记录）


## 5. 可继续扩展方向

- 加“重做本题”按钮
- 错题本与收藏夹
- 做题历史与正确率统计
- 按知识点/难度组卷
- 登录与多用户数据隔离
